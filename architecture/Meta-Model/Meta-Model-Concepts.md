# Meta-Model Concepts

## Purpose
To define the specific instances derived from the universal Engineering Object Model, forming the core conceptual vocabulary of the EOS.

## Core Concepts

| Concept | Definition |
| :--- | :--- |
| **Engineering Domain** | A bounded context of engineering capability. Purpose: Establish clear boundaries. |
| **Artifact** | A tangible, versioned output of engineering work. |
| **Capability** | A discrete, measurable business or technical function. |
| **Workflow** | A repeatable sequence of engineering activities. |
| **Lifecycle** | The end-to-end progression of an engineering object. |
| **Stage** | A distinct phase within a lifecycle. |
| **Gate** | A hard boundary between lifecycle stages. |
| **Review** | A formal evaluation of an object. |
| **AFG (Architecture Freeze Gate)**| A supreme gate locking architectural design before implementation begins. |
| **Verification** | Proof that the system was built correctly. |
| **Validation** | Proof that the correct system was built. |
| **Evidence** | Immutable proof of a state, verification, or decision. |
| **Decision** | A recorded choice between alternatives (e.g., ADR). |
| **Risk** | An identified potential for failure or loss. |
| **Issue** | A realized risk or defect. |
| **Change** | An intentional modification to an artifact or system. |
| **Release** | A packaged deployment of capabilities. |
